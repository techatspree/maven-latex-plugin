TEX4HT(1)                    General Commands Manual                   TEX4HT(1)



NAME
       tex4ht - for authoring hypertext with tex/latex

SYNOPSIS
       tex4ht file[.dvi] [ -ttfm-font-dir ] [ -ihtf-font-dir ] [ -eenv-file ] [
       -dout-dir ] [ -gbitmap-file-ext ] [ -h(e|g|s) ] [ -P(*|filter) ]

       t4ht file[.lg] [ -dout-dir ] [ -mmode ] [ -eenv-file ] [ -hhtf-conf-file
       ] [ -p ] [ -i ] [ -S(*|filter) ]

       ht tex | latex file [ t4ht options ]

       httex|htlatex|httexi file [ options1 ] [ options2 ] [ options3 ]

DESCRIPTION
       This manual page documents briefly the tex4ht and t4ht commands, and
       related scripts, ht, httex, htlatex and httexi.  This manual page was
       written for the Debian GNU/Linux distribution because the original
       program does not have a manual page.  Instead, it has documentation in
       HTML (hypertext) format; see below.  Please refer to the HTML
       documentation for more details or up to date information.

       tex4ht is a highly configurable TeX-based authoring system for producing
       hypertext.  It interacts with TeX-based applications through style files
       and postprocessors, leaving the processing of the source files to the
       native TeX compiler. Consequently, tex4ht can handle the features of TeX-
       based systems in general, and of LaTeX in particular.

       The auxiliary program, t4ht, is used to generate images as required by
       the HTML file, for anything that cannot be rendered using simple HTML
       text and coding.  (The method and programs used for generating the images
       are configurable.)  In addition, it puts together the CSS (Cascade Style
       Sheet) files, and issues system calls for services requested in the
       source (La)TeX files.

       The shell scripts ht, httex, htlatex and httexi are wrappers round the
       other programs, also running tex(1) or latex(1) as required.

OPTIONS
       The following is only a very brief summary of the main program options.
       For more details of the command-line and configuration options, see the
       HTML manual pages (see below).

   General directory/file search options
       Several of the options detailed below relate to setting directory paths
       to find particular kinds of files.  Note the following when specifying
       directories for the programs:

       1.     There must be no space between the option letter (eg, -t) and the
              directory/file name.

       2.     The directory name must end with a trailing slash (/, or /!  - see
              below).

       3.     You can add an exclamation mark (!)  to the end of the directory
              name, to enable sub-directory searching.

       4.     Only one directory can be specified at a time.  To search multiple
              directories, either use the sub-directory searching facility or
              repeat the option.  (The directory search options can be repeated
              as often as required.)

       As well as command-line options, most of these options can (and normally
       should) be set in the configuration file, tex4ht.env.  The format within
       this file is similar, but without using a hyphen before the option
       letter.  See the HTML manual pages for more details on the configuration
       options available.

       In addition, if tex4ht has been compiled with support for the kpathsea
       search library, some options (eg, directories for font files) can be set
       in the global configuration file used by all the TeX-related programs
       that make use of the library.

   Options for tex4ht
       -ttfm-font-dir
              Set directory for TFM font files to tfm-font-dir.

       -ihtf-font-dir
              Set directory for HTF font files (used by tex4ht) to htf-font-dir.

       -eenv-file
              Specify address of the tex4ht configuration file tex4ht.env.

       -dout-dir
              Set the directory for outputting files (.html, .css) to out-dir.
              Note that tex4ht-specific files (.idv, .lg) will still be output
              to the current directory.

       -gbitmap-file-ext
              Set the extension of bitmap images to bitmap-file-ext instead of
              the default .gif.  Note that the extension should begin with a dot
              (.).  Note also that this option only affects images generated
              automatically for symbol fonts, and not images generated with the
              \Picture macro of tex4ht.

       -he    Trace errors and warnings (produces verbose output).

       -hg    Trace groups

       -hs    Trace \specials.

       -Pfilter
              Restrict system calls requested in the source (La)TeX files to
              utilities whose names have filter as a prefix.  Use a single
              asterisk as filter to allow any system calls.

   Options for t4ht
       -dout-dir
              Output files to directory out-dir, instead of to the current
              directory.

       -mmode Create files with access mode as indicated by mode.  The mode
              should be a numeric mode, as used by the chmod(1) command.

       -eenv-file
              Specify address of the tex4ht configuration file tex4ht.env.

       -hhtf-conf-file
              Set the font configuration file to htf-conf-file.  The default is
              htfcss.env.

       -p     Do not convert pictures (ie, images generated with macros like
              \Picture, not characters in symbol fonts that will be
              automatically converted into a graphic file).

       -i     Generate verbose debugging output.

       -Sfilter
              Restrict system calls requested in the source (La)TeX files to
              utilities whose names have filter as a prefix.  Use a single
              asterisk as filter to allow any system calls.

   Options for ht
       ht is a small shell script that is used to run the tex4ht programs.  It
       has two compulsory arguments, as follows:

       1.     The TeX command name to run, usually either tex or latex.

       2.     The input file name, without any extension.

       It may take any additional options, which will be passed on to the call
       to t4ht.

   Options for httex, htlatex and httexi
       httex, htlatex and httexi are shell scripts that can be used for files
       that do not explicitly load the configuration files.  Each script takes
       one mandatory file argument, followed by up to three optional arguments,
       as follows:

       1.     Options for the tex4ht macros (used with tex or latex).

       2.     Directory for tex4ht font files (*.htf).

       3.     Options for the t4ht program.

       See the HTML documentation for more details and examples.

FILES
       /etc/tex4ht/tex4ht.env
              Configuration file for tex4ht.

       /etc/texmf/texmf.cnf
              Global configuration file for all TeX-related programs using the
              kpathsea path searching library.

       /usr/share/texmf/tex4ht/ht-fonts/*.htf
              Hypertext font files used by tex4ht.

       /usr/share/texmf/tex/generic/tex4ht/tex4ht.sty and
       /usr/share/texmf/tex/generic/tex4ht/all4ht/*.4ht
              Macro files used by tex or latex when using tex4ht to generate
              HTML output.

       /usr/share/doc/texmf/tex4ht/mn.html
              top-level HTML file for the manual

SEE ALSO
       The use of the tex4ht macros is documented in the HTML manuals (see file
       name above).

       tex(1), latex(1), amstex(1), kpsewhich(1)

BUGS
       This manual page is not complete.  Users should refer to the HTML manuals
       that form part of the system for fuller details of the configuration
       options and for details of the use of the tex4ht style files.  tex4ht is
       still a work in progress, so bug reports and requests are very welcome:
       contact Eitan Gurari <gurari@cis.ohio-state.edu>.
